# Release Plan Notes

This scaffold does not currently track Ant Design Pro demo releases as direct business requirements. Framework upgrades should be handled as maintenance tasks, not as automatic template rewrites.

## Principles

- Preserve the Umi Max / Ant Design Pro application structure.
- Preserve business pages and local scaffold conventions.
- Do not hand-edit `src/services/ant-design-pro/`.
- Do not reintroduce removed demo pages unless a real scaffold need exists.
- Do not add production dependencies unless the module being built requires them.

## Upgrade Checklist

1. Review upstream Ant Design Pro changes.
2. Compare `config/`, `src/app.tsx`, `src/requestErrorConfig.ts`, and routing behavior.
3. Query antd APIs with `npx antd info <Component> --format json` before modifying antd code.
4. Keep `src/pages/system/example/` working as the CRUD reference.
5. Run the verification baseline:

```bash
npm run lint
npx antd lint ./src --format json
npm run build
npm run test
```

## Build Tool Note

The current default build path is Umi webpack. Re-enabling utoopack requires a fresh compatibility check with Tailwind CSS v4 and a passing `npm run build`.
