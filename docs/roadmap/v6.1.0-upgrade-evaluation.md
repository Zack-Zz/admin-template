# Build And Upgrade Evaluation

This document records the current scaffold decision after simplification.

## Current State

- The project uses Umi Max 4 and Ant Design Pro structure.
- The default `npm run build` uses Umi's webpack bundler.
- The previous explicit `utoopack` config was removed because Tailwind CSS v4 + utoopack failed in this managed environment while evaluating the Tailwind loader.
- Markdown imports are still supported through `config/md-raw-loader.cjs` and `chainWebpack`.
- `esbuildMinifyIIFE: true` is enabled because Umi reported esbuild helper conflicts during webpack production build.

## Upgrade Rule

Do not re-enable `utoopack` only because the upstream template contains it. Re-evaluate it together with Umi Max, Tailwind CSS, and `@tailwindcss/webpack`, then require `npm run build` to pass before accepting the change.

## Verification Baseline

Before upgrading framework packages, run:

```bash
npm run lint
npx antd lint ./src --format json
npm run build
npm run test
```

If OpenAPI output changes, regenerate with `npm run openapi` and review `src/services/ant-design-pro/` as generated code only.
